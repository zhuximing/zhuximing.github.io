Redis集群架构

2017-07-27
Redis

Redis在互联网大数据平台有着广泛的应用，主要被用来缓存热点数据，避免海量请求压垮数据库，同时可以提升服务节点的响应速度和并发量。随着数据量的增多，由于redis是占用单台物理机或虚机的内存，内存资源是有限的，要动态地扩容缩容，就需要用到redis集群。

集群技术是构建高性能网站架构的重要手段，试想在网站承受高并发访问压力的同时，还需要从海量数据中查询出满足条件的数据，并快速响应，我们必然想到的是将数据进行切片，把数据根据某种规则放入多个不同的服务器节点，来降低单节点服务器的压力。


Redis 集群中内置了 16384 个哈希槽，当需要在 Redis 集群中放置一个 key-value 时，redis 先对 key 使用 crc16 算法算出一个结果，然后把结果对 16384 求余数，这样每个 key 都会对应一个编号在 0-16383 之间的哈希槽，redis 会根据节点数量大致均等的将哈希槽映射到不同的节点。

使用哈希槽的好处就在于可以方便的添加或移除节点。

当需要增加节点时，只需要把其他节点的某些哈希槽挪到新节点就可以了；

当需要移除节点时，只需要把移除节点上的哈希槽挪到其他节点就行了；

内部机制，与我何干，对于我们来说，在新增或移除节点的时候不要让我们先停掉所有的 redis 服务我就谢天谢地了，这点它做到了。

fastDFS工作流程图解

2017-07-25
fastDFS

1：文件上传

1：各个storage向tracker每个5s发送心跳检测，发送（ip，端口，负载）等信息
2：用户向tracker服务器发送上传请求。
3：tracker服务器计算得到当前负载最小的storage，并将（ip，端口）信息反馈给请求上传文件的客户端
4：客户端根据返回信息上传文件到指定storage，并在组内实现同步
5：上传完成,返回文件路径信息
2：文件下载 ![](http://139.199.74.187/blog/wp-content/uploads/2017/07/fdfdfeeeee-300x138.jpg) 
1：各个storage向tracker每个5s发送心跳检测，发送（ip，端口，负载）等信息
2：用户向tracker服务器发送下载请求。
3：tracker服务器根据你的请求路径计算得到当前不忙的storage，并将（ip，端口）信息反馈给请求下载文件的客户端
4：客户端根据返回信息到指定storage下载文件

fastDFS简介

2017-07-25
fastDFS未分类 

fastdfs论坛：[http://bbs.chinaunix.net/forum-240-1.html](http://bbs.chinaunix.net/forum-240-1.html)
1>传统文件存储方式 之低并发
![](http://139.199.74.187/blog/wp-content/uploads/2017/07/2017-07-25_115411-300x158.png)
2>传统文件存储方式 之高并发
![](http://139.199.74.187/blog/wp-content/uploads/2017/07/2017-07-25_124204-300x182.png)
传统方式弊端
1：传统方式文件是存储在项目的某个目录下，如果项目clean 文件丢失
2：tomcat之间文件同步难度大。即使可以同步，同步会影响服务器性能
3：逻辑业务和文件上传，文件下载放在同一服务器，影响业务性能
总之传统方式是小学生干的事
怎么办？
答:单独部署一台服务器，专门干文件上传、文件存储、文件下载这些苦力活。
于是乎我们这么玩
![](http://139.199.74.187/blog/wp-content/uploads/2017/07/2017-07-25_130431-300x180.png)
这里引入了fastDFS服务系统

优点
1：传统服务器的弊端全解决了
2：专门干文件的事情（因为专注所以专一）
缺点：
1：就一台服务器，宕机了怎么办？
于是乎我们决定做集群
![](http://139.199.74.187/blog/wp-content/uploads/2017/07/2017-07-25_131539-300x189.png)
集群解决了单点宕机的问题，至于各fastDFS服务器怎么实现文件同步的这个我们可以不关心
接下来，我们来简单了解下fastDFS
FastDFS是一个开源的轻量级分布式文件系统，它对文件进行管理，功能包括：文件存储、文件同步、文件访问（文件上传、文件下载）等，解决了大容量存储和负载均衡的问题。特别适合以文件为载体的在线服务，如相册网站、视频网站等等。

FastDFS为互联网量身定制，充分考虑了冗余备份、负载均衡、线性扩容等机制，并注重高可用、高性能等指标，使用FastDFS很容易搭建一套高性能的文件服务器集群提供文件上传、下载等服务。

那么fastDFS内部架构是怎么样的呢？请看下图
![](http://139.199.74.187/blog/wp-content/uploads/2017/07/2017-07-25_132050-300x196.png)
服务端两个角色：
Tracker：管理集群(类似于zookeeper，但不是zookeeper)，tracker也可以实现集群。每个tracker节点地位平等。收集Storage集群的状态（ip 、端口、负载）,当然这些信息是Storage主动汇报的。
Storage：实际保存文件
Storage分为多个组，每个组之间保存的文件是不同的。每个组内部可以有多个成员，组成员内部保存的内容是一样的，组成员的地位是一致的，没有主从的概念。

为什么要分组？
答：如果不分组，那么每台Storage存放的数据是一样的，这样导致服务器物理存储很容易遇到瓶颈。
分完组，组间文件可以不一样，这样减轻了 Storage存储压力

组内为什么要多台Storage，而且Storage文件保持同步？
答：防止单节点宕机

分完组，组间文件数据不同步，怎么能保证用户一定能拿到文件数据？
答：文件存储完会返回文件路径，文件路径遵循如下格式
![](http://139.199.74.187/blog/wp-content/uploads/2017/07/xxxx-300x20.jpg)
group1：组名，你去这个组找肯定能找到
M00:磁盘名
/02/44:分级目录结构
后面一长串，文件名，通过源存储服务器IP地址、文件创建时间戳、文件大小、随机数和文件拓展名等信息
组合成，保证文件名唯一性

zookeeper-单机版

2017-07-19
dubbo未分类



安装 Dubbo 注册中心(Zookeeper-3.4.6) 

1:安装虚拟机(无界面启动 vim /etc/inittab  将5改成3)如果想切换到有界面 输入命令startx
2：安装centos6.6系统
3:SecurCRT8.x(远程连接虚拟机的客户端)



dubbo建议使用zookeeper作为服务的注册中心

  • 前提：centos需要搭建java环境（zookeeper本身是java程序）

    参考[centos搭建java环境
](http://139.199.74.187/blog/?p=36)

  • 1:下载zookeeper-3.4.6

    方法一(需要连网)：
    a>:wget http://www-eu.apache.org/dist/zookeeper/zookeeper-3.4.6/zookeeper-3.4.6.tar.gz

方法二：
a>:先下载到本地机器，下载地址：http://www-eu.apache.org/dist/zookeeper/zookeeper-3.4.6/zookeeper-3.4.6.tar.gz
b>:使用secureCRT自带上传工具 打开secureCRT按住快捷键alt+p ,将压缩包拖到secureCRT

注意：最好用root用户登录，然后上传，这时候一般上传到root用户的家目录即/root目录下
3:解压zookeeper安装包到/user/local目录下

1>:tar -zxvf zookeeper-3.4.6.tar.gz -C /usr/local
2>修改zookeeper文件夹名字为zookeeper-alone(后面还要搭建集群版)
   cd /usr/local
   mv zookeeper-3.4.6 zookeeper-alone(该文件夹名字)

  • 4:在zookeeper-alone目录下创建以下目录

    cd /usr/local/zookeeper-alone/
mkdir data(创建data文件夹)
mkdir logs（创建logs文件夹,可以忽略）

  • 5:将 zookeeper-alone/conf 目录下的 zoo_sample.cfg 文件拷贝一份，命名为为 zoo.cfg

    cd /usr/local/zookeeper-alone/conf
cp zoo_sample.cfg  zoo.cfg(拷贝)

  • 6:修改zoo.cfg配置文件

    vim zoo.cfg
在#example sakes这行注释下面加上以下代码
dataDir=/usr/local/zookeeper-alone/data
dataLogDir=/usr/local/zookeeper-alone/logs(忽略)
注意：路径一定不要错了

  • 7:启动zookeeper

    1>切换到zookeeper的bin目录
cd /usr/local/zookeeper-alone/bin   
2>./zkServer.sh start               启动zookeeper
3>./zkServer.sh status              查看zookeeper状态
4>./zkServer.sh stop                关闭zookeeper

  • 8:开放2181 端口（如果你觉着麻烦，干脆关闭防火墙也可，关闭防火墙的命令）
    关闭防火墙（建议）

    /etc/init.d/iptables stop
    chkconfig  iptables off

    开放端口(如果做了上一步骤，本步可以忽略,但是在生产环境必须做这一步骤)
    切换到 root 用户权限，执行以下命令：

    chkconfig iptables on 设置防火墙开机启动

    service iptables start 启动防火墙服务

    编辑/etc/sysconfig/iptables

    vi /etc/sysconfig/iptables

    增加以下 1 行：
    -A INPUT -m state –state NEW -m tcp -p tcp –dport 2181 -j ACCEPT
    重启防火墙：

    service iptables restart

    查看防火墙端口状态：

    service iptables status

Docker介绍

2017-07-02
Docker

一：docker思想：

1:集装箱：将一个镜像整体封装起来，而不是零散存放，一起封装起来的好处是便于快速稳定移植，这跟集装箱很相似

2:存储标准化：以前我们要启动应用程序得先找到它的目录，而存放在docker中的应用程序，你只需通过docker命令就可以控制启动和停止

3:隔离：Docker提供一个完整的文件系统，独立的网络环境等等这些功能，同时还没有如此庞大。

4:快速扩展弹性伸缩：比如双11必须增加多个服务器，如果用传统方式一个个依次部署工作量是很大的，但是docker可以轻松帮你解决这些问题。

二：build ship run

build：构建镜像 ship：从中央镜像仓库下载镜像 run：启动镜像，开启容器。

消息中间件《一》

2017-07-02
服务安装


通过上图分析消息中间件能够实现 服务间解耦、异步调用、处理高并发等功能

一：服务间解耦：

如果不用消息中间件支付服务会依赖其他多个服务，使得藕合度太高，使用中间件后支付服务只需要将数据发送给消息中间件，而无须直接调用其他服务，如此实现解耦。

二：异步调用

没用消息中间件，服务的调用是串行调用的，这会导致效率非常低，用了消息队列后，可以发送给队列进行异步处理，而且无需其他服务必须是运行状态。

三：处理高并发：

举12306网站来说，在高峰期抢票的时候，你不能只处理前100个请求，你要处理所有的请求，这对系统是很大的挑战，用消息队列，我们可以将用户数据发送给消息队列，消息队列一次处理，这样的话，系统就可以接纳海量请求。

centos搭建nodeJs运行环境

2017-06-21
NodeJs

环境需求：
GCC 4.8.1或者更新
G++4.2或者更新
Python2.6或者2.7

一：检测当前centos系统的版本，命令如下：
cat /etc/redhat-release 我的centos版本是6.5

二：检查gcc，gcc-c++命令如下：
rpm -q gcc rpm -q gcc-g++
如果没有安装可以通过yum -y install gcc-c++ kernel-devel 我的centos已经装了gcc但是没有装gcc-c++

三：检查python，命令如下：
python -V 我的python是2.6.6

是不是觉得步骤多恨麻烦，当然你还可以通过以下一条命令，搞定nodejs运行环境 简单粗暴

yum -y update && yum -y groupinstall “Development Tools”

开始安装nodejs

1:下载nodejs源码包 通过下面命令：

wget https://nodejs.org/dist/v6.11.0/node-v6.11.0.tar.gz 我是安装在/usr/src目录下面的

2:解压：tar -xf node-v6.11.0.tar.gz

3:配置 ./configure

4: 编译:make

5:安装 make install

然而第四步一直报错，估计是gcc版本太低（我的是太低了不想升级gcc），这里我放弃通过源码编译安装，直接下载编译后的文件进行安装

1:wget https://nodejs.org/dist/v6.11.0/node-v6.11.0-linux-x64.tar.xz

2:配置node环境变量

  • vi /etc/profile.d/node.sh 当系统启动会扫描profile.d文件夹下面所有的.sh 文件
  • NODE_HOME=/usr/local/node-v6.11.0-linux-x64 node编译后二进制文件位置
    PATH=$NODE_HOME/bin:$PATH 在linux系统拼接环境变量路径用冒号而不是window系统的分号 $PATH是引用已存在的环境变量路径。
  • export NODE_HOME PATH 导出NODE_HOME和拼接后的PATH这样才能在系统的任何位置都可以访问
  • :wq保存
  • source /etc/profile 重新加载所有的在profile.d文件夹下面的.sh文件，这样上面配的环境变量才会生效
    3:验证:node -v 出现版本号即可
