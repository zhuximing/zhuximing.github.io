HashSet怎么判断两个对象是否重复的

2017-06-13
JAVA_WEB未分类

我们都知道hashset特点就是无序不能重复，那么hashset怎么做到不能重复的呢？尤其当集合里面存放的是对象。

查看了JDK源码，发现HashSet竟然是借助HashMap来实现的，利用HashMap中Key的唯一性，来保证HashSet中不出现重复值。具体参见代码：


/由此可见HashSet竟然是借助HashMap来实现的/
public HashSet() {
        map = new HashMap<>();
 }

/通过map判断key是否重复的方式来判断元素是否重复/
 public boolean contains(Object o) {
        return map.containsKey(o);
    }

/set中的元素是存放在map中的key/
public boolean add(E e) {
return map.put(e, PRESENT)==null;
}

那么map又是怎么保证key的唯一的呢?请看源码

public V put(K key, V value) {
    if (key == null)
        return putForNullKey(value);
    int hash = hash(key.hashCode());//调用key的hashcode方法
    int i = indexFor(hash, table.length);
    for (Entry<K,V> e = table[i]; e != null; e = e.next) {
        Object k;
        if (e.hash == hash && ((k = e.key) == key || key.equals(k))) {
        V oldValue = e.value;
        e.value = value;
        e.recordAccess(this);
        return oldValue;
        }
    }
}

通过上面代码不难发现HashMap中的Key是根据对象的hashCode() 和 euqals()来判断是否唯一的。也就是说set也是根据对象的hashCode() 和 euqals()来判断是否重复的

结论：当要保存对象与set集合中的某个对象的hashcode方法的返回值相等 同时 被保存对象.equals（比较对象）返回值为true则认定两个对象重复 无法重复添加

现在我们可以来重写对象的hashcode方法和equals方法来控制对象是否重复 代码如下:

package com.glass;

import com.hobby.userCenter.domain.Student;

public class Student {

private int stuId;

public int getStuId() {
    return stuId;
}

public void setStuId(int stuId) {
    this.stuId = stuId;
}

@Override
public int hashCode() {
    // TODO Auto-generated method stub
    return stuId;
}

@Override
public boolean equals(Object obj) {
    // TODO Auto-generated method stub
    Student s=(Student)obj;   
    return  stuId==s.getStuId();
}

}


测试:

 public static void main(String[] args) {
        Student s1=new Student();
        s1.setStuId(1);
        Student s2=new Student();
        s2.setStuId(1);
        HashSet set=new HashSet();
        set.add(s1);
        set.add(s2);
        System.out.println(set.size());
    }

结果输出为1 。id都一样 ，在内存里面是两个不同的对象 但是结果set判定这两个对象相等，明白原理才能明白结果，我算明白了！！

centOS安装redis-单机

2017-06-05
Redis未分类

一：系统安装环境

由于redis是c语言开发的，所以依赖gcc环境，如果系统没有gcc环境可以通过以下命令安装
gcc --version  //查看gcc版本信息
yum install gcc-c++如果提示下载请选择y
**二：下载redis压缩包并上传到centos服务器** 
下载地址：https://redis.io/
如果是window用户，请使用secureCRT上传压缩包
**三：解压redis命令如下** 
tar -zxvf redis-3.2.9.tar.gz
**四：编译:(编译需要gcc环境)** 
[root@localhost ~]# cd redis-3.0.0               //切换到源码包
[root@localhost redis-3.0.0]# make              //编译源码


**五：安装**
[root@localhost redis-3.0.0]# make PREFIX=/usr/local/redis install 
 (/usr/local/redis 为安装目录)安装成功后在redis下有个bin目录在bin下有如下可执行文件
![](http://139.199.74.187/blog/wp-content/uploads/2017/06/屏幕快照-2017-06-05-下午9.33.28-300x39.png)
六：将redis源码包中的reids.conf拷贝一份到/usr/local/redis 
[root@localhost redis-3.0.0]# cp redis.conf /usr/local/redis
  七：修改redis.conf 配置文件 
vi /usr/local/redis/redis.conf  

![](http://139.199.74.187/blog/wp-content/uploads/2017/06/屏幕快照-2017-06-05-下午9.53.34-300x130.png)

将daemonize对应的值改为yes   （守护进程即后台启动）
还可以修改port端口号  默认端口号是6379
八：切换到redis安装目录 的bin目录下，启动redis服务命令如下 
cd /usr/local/redis/bin
[root@localhost bin]# ./redis-server ../redis.conf
九：查看redis服务启动情况 
ps aux|grep redis-server
十：使用redis-cli(redis客户端)操作redis 输入redis-cli建立连接 
./redis-cli -h远程ip  －p端口     
通过info命令查看redis服务器当前信
十一：关闭redis服务 
1:kill 进程号
2:使用redis-cli 执行shutdown命令
十二：给redis服务器设置密码 
**1、初始化Redis密码：**
   在配置文件redis.conf中有个参数： requirepass 这个就是配置redis访问密码的参数；
   比如 requirepass test123；
   （Ps:需重启Redis才能生效）redis的查询速度是非常快的，外部用户一秒内可以尝试多大150K个密码；所以密码要尽量长（对于DBA 没有必要必须记住密码）

**2、不重启Redis设置密码：**
   设置密码（先无密码登陆，再设置）
     redis 127.0.0.1:6379> config set requirepass zhu
   查询密码：(因为该了密码，会提示操作被拒绝)
   redis 127.0.0.1:6379> config get requirepass
   (error) ERR operation not permitted

   密码验证：（使用设置的密码进行登陆）
   redis 127.0.0.1:6379> auth zhu

centOS 配置java环境

linux

centOS 配置java环境

**注意：统一使用jdk1.7版本,64位
**

1>:将jdk压缩包上传至虚拟机（使用secureCRT工具 快捷键alt+p 直接拖拽，简单粗暴）
2> tar -zxvf jdk-7u25-linux-x64.tar.gz -C /usr/local 
 -C代表解压到目标路径 C要大写

 
5>配置java环境变量 ** 
  1:touch /etc/profile.d/java.sh
  2>** **vi /etc/profile.d/java.sh       当系统启动会扫描profile.d文件夹下面所有的.sh 文件                         **
  **3>JAVA_HOME**=**/usr/local/jdk1.7.0_25** 
  4>**PATH=$JAVA_HOME/bin:$PATH       $PATH是引用已存在的环境变量路径**
  **5>export  JAVA_HOME PATH**         **导出JAVA_HOME 这样才能在系统的任何位置都可以访问**
 ** 6>:wq                                 保存 
  7>source /etc/profile 
             重新加载所有的在profile.d文件夹下面的.sh文件，这样上面配的环境变量才会生效
**
6>测试 通过java -version 或者echo $PATH 查看打印信息即可验证你的环境变量配置是否成功
