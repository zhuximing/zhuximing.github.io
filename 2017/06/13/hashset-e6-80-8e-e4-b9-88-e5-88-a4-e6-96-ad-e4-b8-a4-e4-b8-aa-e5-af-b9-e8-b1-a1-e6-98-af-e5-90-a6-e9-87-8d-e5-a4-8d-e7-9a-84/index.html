HashSet怎么判断两个对象是否重复的

2017-06-13
JAVA_WEB未分类

我们都知道hashset特点就是无序不能重复，那么hashset怎么做到不能重复的呢？尤其当集合里面存放的是对象。

查看了JDK源码，发现HashSet竟然是借助HashMap来实现的，利用HashMap中Key的唯一性，来保证HashSet中不出现重复值。具体参见代码：


/由此可见HashSet竟然是借助HashMap来实现的/
public HashSet() {
        map = new HashMap<>();
 }

/通过map判断key是否重复的方式来判断元素是否重复/
 public boolean contains(Object o) {
        return map.containsKey(o);
    }

/set中的元素是存放在map中的key/
public boolean add(E e) {
return map.put(e, PRESENT)==null;
}

那么map又是怎么保证key的唯一的呢?请看源码

public V put(K key, V value) {
    if (key == null)
        return putForNullKey(value);
    int hash = hash(key.hashCode());//调用key的hashcode方法
    int i = indexFor(hash, table.length);
    for (Entry<K,V> e = table[i]; e != null; e = e.next) {
        Object k;
        if (e.hash == hash && ((k = e.key) == key || key.equals(k))) {
        V oldValue = e.value;
        e.value = value;
        e.recordAccess(this);
        return oldValue;
        }
    }
}

通过上面代码不难发现HashMap中的Key是根据对象的hashCode() 和 euqals()来判断是否唯一的。也就是说set也是根据对象的hashCode() 和 euqals()来判断是否重复的

结论：当要保存对象与set集合中的某个对象的hashcode方法的返回值相等 同时 被保存对象.equals（比较对象）返回值为true则认定两个对象重复 无法重复添加

现在我们可以来重写对象的hashcode方法和equals方法来控制对象是否重复 代码如下:

package com.glass;

import com.hobby.userCenter.domain.Student;

public class Student {

private int stuId;

public int getStuId() {
    return stuId;
}

public void setStuId(int stuId) {
    this.stuId = stuId;
}

@Override
public int hashCode() {
    // TODO Auto-generated method stub
    return stuId;
}

@Override
public boolean equals(Object obj) {
    // TODO Auto-generated method stub
    Student s=(Student)obj;   
    return  stuId==s.getStuId();
}

}


测试:

 public static void main(String[] args) {
        Student s1=new Student();
        s1.setStuId(1);
        Student s2=new Student();
        s2.setStuId(1);
        HashSet set=new HashSet();
        set.add(s1);
        set.add(s2);
        System.out.println(set.size());
    }

结果输出为1 。id都一样 ，在内存里面是两个不同的对象 但是结果set判定这两个对象相等，明白原理才能明白结果，我算明白了！！
